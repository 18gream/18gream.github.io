Enhancement One: Software Design & Engineering

This artifact was previously designed in CS-360. The application is designed to track weight, modify entries, and receive notifications. The artifact was selected for this category because it showcases design and development aspects to applications.

Project repo: artifact1 GitHub Repository

Reflection Narrative

This artifact demonstrates software development and engineering skills across multiple areas. These include user account management by creating and logging in using SQLite, data storage and retrieval using database helpers, SMS notifications, and permission handling through Android APIs. The artifact was enhanced in various ways. To start, the login and registration page was enhanced through adding proper validation and feedback for user credentials. Elements were added to ensure duplicate username and passwords were prevented by checking the database before inserting. Last, findViewById was not matching XML ID, therefore that was updated too. Next, the layout files were cleaned up to align with proper UI elements (ConstraintLayout and LinearLayout). Issues with incorrect and missing IDs like editWeight and editDate were solved. Enabling of updating weight entries was adding by using dialog in RecyclerView adapter as well. For the weight entry functionality, I connected the weight entry system with the local database and included addWeightData, updateWeightData and deleteWeightData with UI refresh. There originally was no dialog field to add weight data, so that was added in the DatabaseHelper. I also included deletion and editing of weight entries through dialogs. For the SMS notifications, I fixed logic where the message sending did not align with user input regarding the phone number. There was a resource linking failed error due to improper XML attributes, which was resolved. I added clear and concise comments within the code as some did not have any comments at all. Overall, I believe I met the course outcomes that I planned to meet with the first enhancement. I improved the code maintainability and quality, used engineering best practices and used modern tools and libraries. I also implemented CRUD operations by allowing users to add, update and delete entries. I also improved the input validation. I do not have any updates to my outcome coverage plan at this time. During the enhancement, I learned more about debugging and how to fix improper XML and Java connections. I enhanced my understanding about working with Android’s permission system and how it affects runtime behavior. I increased my knowledge of UI responses and how different emulators or devices behave differently. Some challenges I faced were improper attributes and missing references causing the error “resource linking failed.” I also faced emulator performance issues causing lags and crashes. I dealt with crashes that were caused by XML resources and views that were not declared right. Lastly, consistency across layout files and Java code was not there, and issues with ensuring consistency occurred.
